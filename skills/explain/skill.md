1. **What it does** — one paragraph summary
2. **How it works** — step-by-step walkthrough of the key logic
3. **Why** — design decisions, trade-offs, or patterns used (if apparent)
4. **Dependencies** — what it calls, what calls it

Keep it concise. Adjust depth to complexity — don't over-explain simple code. Use the reader's likely skill level as context (if they ask about a basic function, keep it beginner-friendly).
